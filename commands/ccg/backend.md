# Backend - 后端专项开发

后端专项开发，Codex 主导分析和审查。

## 使用方法

```bash
/ccg:backend <后端任务描述>
```

---

## 执行方式

**执行方式**：Task 调用代理

**代理**：`backend-agent`

**调用**：
```
Task({
  subagent_type: "backend-agent",
  prompt: "$ARGUMENTS",
  description: "后端专项开发"
})
```

---

## 工具集

**代理调用的工具**：
- 代码检索：`mcp__ace-tool__search_context`
- 外部模型：Codex（后端权威）+ Gemini（辅助参考）
- 用户确认：`mcp______zhi`
- 知识存储：`mcp______ji`

**详细说明**：参考 [backend-agent.md](../../agents/ccg/backend-agent.md)

> 仅列关键工具，完整清单见代理文件

## 执行流程（6 阶段）

backend-agent 将执行以下 6 阶段工作流：

1. **增强需求**（可选）— 结构化技术需求
2. **检索上下文** — 获取相关代码和配置
3. **Codex 分析** — 技术分析和架构规划
4. **实施** — 代码开发
5. **Codex 审查** — 代码审查和优化
6. **评审** — 最终评估

---

## 关键规则

1. **Codex 后端意见可信赖**
2. **Gemini 后端意见仅供参考**
3. 外部模型对文件系统**零写入权限**
4. Claude 负责所有代码写入和文件操作

---

## 完成后推荐

后端开发完成后，可能需要的后续操作：

- `/ccg:test` — 生成测试用例
- `/ccg:review` — 代码审查
- `/ccg:commit` — 提交代码变更
